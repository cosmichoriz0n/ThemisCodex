4. Barcode-Based Inventory Tracking

4.1 Barcode architecture

  ------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  Component          Technology                                         Details
  ------------------ -------------------------------------------------- --------------------------------------------------------------------------------------------------
  Barcode format     Code128 + QR (both per item)                       Code128 for USB scanner speed. QR for mobile camera. Both generated on item creation.

  Generation         bwip-js (server-side Node.js)                      Runs in Next.js API route. Never client-side. Returns PNG buffer.

  Label printing     pdf-lib (server-side)                              A4 sheet (multiple labels) or 58mm thermal label. Browser print dialog.

  USB scanner        HID keyboard emulation                             Plug-and-play. No driver needed. Scanner input detected as fast keystroke burst ending in Enter.

  Mobile scanning    html5-qrcode (browser camera)                      Works on Android Chrome and iOS Safari. No app install needed. PWA page.

  Asset tag format   {CODE}-{YYYY}-{SEQUENCE}                           Example: LM-2026-001234 | MP-2026-000089 | UPIS-2026-000001

  Label contents     Asset tag, name, category, location, Code128, QR   QR links to IMIS item record URL for instant lookup
  ------------------------------------------------------------------------------------------------------------------------------------------------------------------------

4.2 Barcode scanning workflows

-   Receiving (stock-in): scan barcode on supplier delivery note - system pre-fills item fields - inspector confirms quantity and condition

-   Issuance: scan requester ID card + item barcode - system does MIMS member lookup - creates issuance record with member link

-   Physical count/audit: scan all items in a location - system compares to expected on-hand - flags discrepancies in variance report

-   Return: scan item barcode - system retrieves original issuance record - one-click return transaction

-   Disposal: scan item barcode - system shows item details + disposal form - approver digitally signs off

5.2 CAS2000 - NEA Chart of Accounts journal mapping

  -------------------------------------------------------------------------------------------------------------------------------------------------------
  Journal type           Trigger                         Debit account                       Credit account                  Amount
  ---------------------- ------------------------------- ----------------------------------- ------------------------------- ----------------------------
  COGS                   Stock-out (issuance)            5110 Cost of Goods Sold             1540 Materials and Supplies     Transaction total amount

  Adjustment             Manual stock adjustment         5120 Inventory Adjustment Expense   1540 Materials and Supplies     Adjustment value

  Disposal write-off     Asset disposed                  5130 Loss on Disposal               1920 Utility Plant in Service   Asset net book value

  UPIS depreciation      Annual (Jan 1 n8n workflow)     5310 Depreciation Expense - UPIS    1990 Accumulated Depreciation   Annual depreciation amount

  Stock-in (receiving)   Receiving goods from supplier   1540 Materials and Supplies         2110 Accounts Payable           PO total amount

  Journal reversal       Void or error correction        Original credit account             Original debit account          Same amount (reversed)
  -------------------------------------------------------------------------------------------------------------------------------------------------------

  Confirm exact NEA COA account codes with the cooperative Finance Officer before Sprint 7. The codes above are based on NEA USOA standards but individual cooperatives may have sub-accounts.
